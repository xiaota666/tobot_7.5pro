pro里面的文件都有什么？

● pro根目录存放菜单脚本
● pro/common存放pro脚本主要逻辑
● pro/config存放自定义配置
● pro/lib存放脚本模板参数和参数说明
● pro/log存放脚本执行日志
● pro/module存放脚本模板配置
● pro/mouse存放鼠标脚本文件

什么是菜单脚本？

是指pro根目录下的脚本，也是土拨鼠能直接读取到的脚本，例如：A阿尔福那酒店.lua。这类脚本可以自己重命名、修改配置、切换目录等

菜单脚本切换目录是什么意思，有什么用？

菜单脚本能够存放在土拨鼠scripts目录下的任何目录下，可以方便自定义一键读取一个目录下的所有菜单脚本，例如：
● 在土拨鼠scripts目录下新建个超能任务文件夹，在超能任务文件夹内存放pro下的所有[超能任务]菜单脚本，这样在你想要找指定超能任务时能快速找到
● 在土拨鼠scripts目录下新建个转点打的脚本文件夹，在转点打的脚本文件夹内存放自定义的Z转点重置副本次数、N扭曲的世界树-自改打1次、G怪物树山山谷-自改打1次等，这样使用土拨鼠一键添加目录下的所有LUA脚本功能就能便捷添加需要打的所有LUA脚本

我想修改pro/module下的文件可以吗？

不建议修改。更新时会更新pro/module下的文件，建议在此副本的菜单脚本文件内修改

[new][副本脚本]我想新增物品记录怎么设置？

方法一（单个修改）：
● 选择你要修改的菜单脚本文件右键记事本打开
● 在最后一行代码dsm.doDungeonScripts(mod)之前加入或修改以下代码： 

mod.itemRecordList = {"斗兽场 小型 体力 药水","斗兽场 小型 复合 药水"}
其中"斗兽场 小型 体力 药水"、"斗兽场 小型 复合 药水"为需要监测的物品名称，可添加多个

方法二（全部修改）：
● 打开pro/config目录下的副本脚本全局配置文件，右键记事本打开
● 在第一行代码return {和最后一行代码}之间加入或修改以下代码： 

itemRecordList = {"斗兽场 小型 体力 药水","斗兽场 小型 复合 药水"},
其中"斗兽场 小型 体力 药水"、"斗兽场 小型 复合 药水"为需要监测的物品名称，可添加多个

[new][通用脚本]最高权限配置是什么意思，如何设置？

用于全局配置下，单个脚本实现最高权限配置。例如：全局设置副本门口等待20秒，可以使用最高权限配置实现单个脚本门口等待10秒

副本脚本、超能副本脚本：
● 选择你要修改的菜单脚本文件右键记事本打开
● 在最后一行代码dsm.doDungeonScripts(mod)之前加入或修改以下代码： 

local rootMod = {}
rootMod.mapCount = 5
其中rootMod.mapCount为执行副本的次数，与mod.mapCount操作相似，多个其他配置也需要在最后一行代码之前，local rootMod = {}之后加

● 将最后一行代码dsm.doDungeonScripts(mod)修改为以下代码： 

dsm.doDungeonScripts(mod,rootMod)

非副本超能脚本、图鉴脚本：
● 选择你要修改的菜单脚本文件右键记事本打开
● 在最后一行代码fsm.doFieldScripts(mod)之前加入或修改以下代码： 

local rootMod = {}
rootMod.mapCount = 5
其中rootMod.mapCount为执行副本的次数，与mod.mapCount操作相似，多个其他配置也需要在最后一行代码之前，local rootMod = {}之后加

● 将最后一行代码fsm.doFieldScripts(mod)修改为以下代码： 

fsm.doFieldScripts(mod,rootMod)

[图鉴脚本]我想设置指定时间结束、倒计时结束、永远不结束脚本该如何设置？

● 选择你要修改的菜单脚本文件右键记事本打开
● 在最后一行代码fsm.doFieldScripts(mod)之前加入或修改以下代码：

不限时模式示例： 

mod.timeLimitType = 1

倒计时模式示例： 

mod.timeLimitType = 2
mod.timeLimitData = 10
其中10为倒计时10分钟后结束脚本，单位分钟

到点模式示例： 

mod.timeLimitType = 3
mod.timeLimitData = {16,15}
其中16为结束小时，15为结束分钟，总体表现为到16点15分结束脚本

[通用脚本]如何在脚本结束后切换指定装备？

end
其中攻击装为pro/config目录下的文件，自定义时请确保文件名和参数相同 修改中文代码请注意编码问题
